# Maintenance Rules

## Goal

Evolve agent behavior without creating context bloat.

## Update Trigger

- Add or edit a rule only when a mistake repeats at least twice, or when impact is high.
- If a rule is one-off or task-specific, keep it in a contract, not global rules.
- Promote retro notes only if they are generalizable and can be expressed in 1-2 lines.

## Update Format

- Keep each new rule to one clear instruction plus one short rationale.
- Prefer updating existing rules over creating new files.
- Avoid overlapping rules that say the same thing differently.

## Review Cadence

- Monthly: remove stale rules, merge duplicates, and resolve contradictions.
- After major workflow changes: re-check completion gates and skill usage guidance.
- Keep completed contracts as archive material, not active default context.
